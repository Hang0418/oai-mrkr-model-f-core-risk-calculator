# MRKR external validation processing plan

## Role in the project

MRKR is used as an external real-world validation cohort for an OAI-derived transport model. It should not be treated as a direct replica of OAI Model E because MRKR does not contain WOMAC function/stiffness, BMI, manual JSN grades, or scheduled OAI-like visits.

The OAI primary model remains:

- **Model E**: baseline symptoms + baseline KL/JSN + 0-24 month symptom changes + 0-24 month structural changes.

The MRKR validation model is:

- **Model F: OAI-MRKR transport model**: age + sex + knee side + landmark pain severity + baseline KL grade + KL change.

## MRKR variable mapping

| Model F construct | OAI source | MRKR source | Processing decision |
| --- | --- | --- | --- |
| Age | `subject_v00age_num` / 24-month age proxy | `age_at_exam` | Use MRKR landmark radiograph age. |
| Sex | `enrollee_p02sex_num` | `sex` | Female coded 1, male coded 0. |
| Knee side | `side` | `laterality` / KL side | Harmonized to right/left. |
| Pain severity | WOMAC pain at 24 months | `pain_score` | OAI WOMAC pain divided by 2 to approximate a 0-10 scale; MRKR uses nearest 0-10 knee pain score within 180 days of landmark. Bilateral pain is mapped to both knees. |
| KL grade | baseline KL | `L_KLG_inference` / `R_KLG_inference` | MRKR uses model-inferred KL grade. |
| KL change | 24-month KL minus baseline KL | landmark KL minus baseline KL | Baseline is earliest no-hardware side-specific KL radiograph; landmark is closest to 24 months within 18-36 months. |
| Primary outcome | target-knee KR/TKA | `arthroplasty` in image metadata | Side-specific arthroplasty hardware first observed after landmark. |
| Sensitivity outcome | target-knee KR/TKA | CPT codes | Patient-level knee arthroplasty CPT after landmark; not side-specific, therefore sensitivity only. |
| Follow-up time | months after 24-month landmark | de-identified dates | Primary time is landmark to hardware event or last target-knee radiograph. |

## CPT outcome definition

Surgical knee arthroplasty CPT codes used for the sensitivity outcome:

- `27438`, `27440`, `27442`, `27445`, `27446`, `27447`, `27486`, `27487`.

Primary TKA CPT sensitivity additionally flags `27447`.

Anesthesia code `01402` is not used as a surgical outcome.

## Current validation interpretation

Model F preserves useful risk ranking in MRKR, but OAI absolute risk is not directly calibrated to MRKR. This is expected because MRKR is a clinical imaging dataset enriched for arthroplasty and hardware follow-up, while OAI is a scheduled research cohort.

The manuscript should therefore describe MRKR as external transport validation with recalibration needs:

- Report discrimination as the main transportability result.
- Report calibration drift explicitly.
- Treat CPT validation as sensitivity because CPT lacks knee side.
- Avoid claiming that MRKR fully validates OAI Model E; it validates a reduced common-variable Model F.
